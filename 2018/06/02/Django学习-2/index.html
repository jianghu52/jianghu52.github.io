关于pycharm社区版

pycharm 当时买了一个正版，可是只能用一年。现在已经3年过去了，实在是太贵，不想买了，就开始用社区版了。结果发现社区版在使用django上有一些坑。

社区版本身是不支持django的，所以需要我们自己手动配置。（以下是pycharm已经配置完enve之后的情况）
1.下载django（现在已经默认是2.0了，这跟很多教程又有区别了。）
2.通过命令开始生成django工程

django-admin startproject 你的工程名

注意，这个地方的工程名跟你实际的项目名还是有差别的，一个工程下面可以跑N个项目，所以这个名字一般不会有强制要求，你怎么起都行。

3.生成django项目

python manage.py startapp 你的项目名

这个项目名最好起的规范一点，因为这个项目名有时候回暴露在url下。你起的太随意比较low

4.manage自检

python manage.py migrate

这个命令执行之后如果都是ok的话。就可以了。
PS：还有两个不是坑的坑。1.自己生成的文件夹里面是不包含的templates文件夹的。我到现在才知道原来这个文件夹是pycharm自己生成的。2.默认的django是链接sqllite3的。所以如果要连接mysql的话，需要手动配置。

5.debug配置
pycharm需要手动配置debug环境。也很简单
run-editconfiguration
打开之后，点击左上角的绿色+号。新建一个。选择python
之后名字随意起。
主要是两个部分

  1. script path：一直选择到manage.py的位置。
  2. Parameters：runserver
