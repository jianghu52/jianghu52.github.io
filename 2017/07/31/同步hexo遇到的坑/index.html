同步hexo遇到的坑

虽然现在看起来，同步的步骤很简单，但是之前碰壁了N次，遇到了好几个坑。记下来，以便以后能跳过去。

1. npm换源

前一台电脑的宽带是联通的，npm虽然慢，好歹还能用。这次的环境宽带是移动的，虽然号称100M。但是用起来完全没有联通的50M的流畅，时不时就断流，要不是我这个小区只有这一个选择，打死我都不用移动的宽带。因为npm没速度，所以准备换淘宝的源。
随便从网上找了一个命令

$ npm install -g cnpm –registry=https://registry.npm.taobao.org

当时也没看，觉得挺对，就用上了。结果一直连不上，我还以为是淘宝的源也不对。后来仔细一看，原来是用cnpm替代了npm。所以如果用上面这条命令，如果在遇到教程，就应该直接自动把npm替换成cnpm。

或者直接替换npm的源。

npm config set registry https://registry.npm.taobao.org

用这个命令之后，后面可以直接使用npm命令。

2. lf will be replaced by crlf

编译文章的时候，报以上的错误。这个错误在我初始的电脑上安装的时候没有报错，当时那台电脑也是win7.至于为什么没报错，我也不知情。但是这个错误其实是跟git有关。而且只出现咋windows系统上。修改方式也简单。

$ rm -rf .git
$ git config –global core.autocrlf false

注意，一定要先删除.git文件之后，再配置git config。不然不好使。

3.hexo deploy not found git

我个人是执行了

npm install hexo-deployer-git –save

就ok了，但是看网上的意思，这个跟没有配置git的环境也有关系。但是因为我配置了，所以这一步坑不是很大。顺手写上git的配置

$ git config –global user.name “runoob”

$ git config –global user.email test@runoob.com

生成ssh

ssh-keygen -t rsa -C “test@runoob.com”

默认保存路径

C:\Users\Administrator.ssh

4.could not read Username for ‘https://github.com‘

这是我最奇怪的一个错误
我原来的配置信息

deploy:   
    type: git   
    repo: https://github.com/jianghu52/jianghu52.github.io
    branch: master

我以为这个是对的，但是在我现在的机器上就是不好使。
改成这样

deploy:
    type: git
    repository: https://jianghu52@github.com/jianghu52/jianghu52.github.io
    branch: master

竟然就对了。我很疑惑到底是什么导致这个差异，可惜到现在也没调查出来。

以上就是遇到的坑。
总结一下大概的流程。

  1. git，node.js 是必须的。
  2. copy原来的scaffolds，source，themes，.gitignore，_config.yml，package.json六个文件，前三个是文件夹，后三个是单文件

  3. 在copy后的文件夹里面执行hexo安装程序

    npm install hexo-cli -g

    注意，最好是换源之后再执行，不然过不去的可能性很大。

  4. hexo server测试过后，直接hexo g，再hexo d。如果一切顺利，那么你应该可以在github上看到你新的blog了。
