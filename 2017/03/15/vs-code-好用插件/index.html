vs code 好用插件

  • vscode-icons

    绝对不能错过的插件。默认图标完全不能看，一个html文件跟.vb文件放一起，谁能一眼看明白？所以这个插件一定要有。

  • Numbered Bookmarks

    个人习惯用书签，选用编辑器的时候，第一要看的就是书签功能是否强大。这也是我弃用很多编辑器的主要原因。个人喜欢带数字的标签，这样定位准确一些，之前用的vs2015，默认书签那个难受啊。各种跳转。要不是看在他的debug功能很逆天的情况下，我早就弃了。

  • Code alignment

格式强迫症的福音。直观一点，举个栗子。

转化之后

对于处女座来说，不这样代码简直不能出去见人。更何况这么写的一大好处就是但凡赋值的地方有点什么不一样的错误，一眼就看得出来。

  • vscode-php-intellisense

    很牛逼的一个插件，但是也很坑爹。配置很奇葩。
    牛逼的地方在于基本上这一个插件能干很多事情，坑爹的地方在于他需要php7的支持。那么像我等还在用5.x系列的屌丝，是不是就不能用了呢。当然不是啦。要怎么用呢。听我道来。

首先，要说明的是我们如果装的是xampp系列，那么我们的环境变量里面是没有php环境的。这就导致了vscode本身是无法识别php环境。所以我们需要配置一个php.validate.executablePath的环境变量。也就是
